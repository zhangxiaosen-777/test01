2.1 Git 基础 - 获取 Git 存储库

如果你只能阅读一章来开始使用 Git，就是这样。 本章涵盖了你完成 Git 最终将花费时间做的绝大多数事情所需的每个基本命令。 在本章结束时，您应该能够配置和初始化存储库、开始和停止跟踪文件以及暂存和提交更改。 我们还将向您展示如何设置 Git 以忽略某些文件和文件模式，如何快速轻松地撤消错误，如何浏览项目历史记录并查看提交之间的更改，以及如何从远程存储库推送和拉取。

获取 Git 存储库

通常通过以下两种方式之一获取 Git 存储库：

  1. 您可以获取当前不受版本控制的本地目录，并将其转换为 Git 存储库，或者

  2. 您可以从其他位置克隆现有的 Git 存储库。

无论哪种情况，您最终都会在本地计算机上获得一个 Git 存储库，随时可以工作。

初始化现有目录中的存储库

如果你的项目目录当前不受版本控制，并且你想开始使用 Git 控制它，你首先需要转到该项目的目录。 如果您从未这样做过，则根据您运行的系统，它看起来会有所不同：

对于 Linux：

$ cd /home/user/my_project

对于 macOS：

$ cd /Users/user/my_project

对于 Windows：

$ cd C:/Users/user/my_project

和类型：

$ git init

这将创建一个新的子目录，该子目录包含所有必需的存储库文件，即 Git 存储库框架。 此时，尚未跟踪项目中的任何内容。 请参阅 Git 内部，了解有关刚刚创建的目录中包含哪些文件的详细信息。.git.git

如果要开始对现有文件（而不是空目录）进行版本控制，则可能应该开始跟踪这些文件并执行初始提交。 您可以通过一些命令来指定要跟踪的文件，然后使用：git addgit commit

$ git add *.c
$ git add LICENSE
$ git commit -m 'Initial project version'

我们将在一分钟内介绍这些命令的作用。 此时，你有一个 Git 存储库，其中包含跟踪的文件和初始提交。
